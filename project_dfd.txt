# Data Flow Diagrams (DFD) – Integrated Charity Crowdfunding, Disaster Management & Food Donation System

This file contains Mermaid code for DFD Level 0, Level 1, and Level 1.1–1.n (by role/module) as per project_spec.txt. Copy each code block into a Mermaid-supported editor (e.g. Mermaid Live, GitHub, Notion) to render the diagram.

---

## DFD Level 0 (Context Diagram – Entire Project)

The system is shown as a single process. External entities are Donor, Campaign Creator/NGO, Volunteer, Admin, and Guest. All interactions with the system flow through this one process.

```mermaid
flowchart LR
    subgraph External_Entities
        DONOR[Donor]
        CREATOR[Campaign Creator / NGO]
        VOL[Volunteer]
        ADMIN[Admin]
        GUEST[Guest / Public]
    end

    subgraph System
        SYS(("Integrated Charity & Disaster Relief System"))
    end

    DONOR -->|"Register, Login, Donate money/food, View history"| SYS
    SYS -->|"Campaigns, Donation confirmations, History"| DONOR

    CREATOR -->|"Register, Login, Create/Update campaigns, View funds"| SYS
    SYS -->|"Campaign status, Donations received"| CREATOR

    VOL -->|"Register, Login, View tasks, Accept/Update task status"| SYS
    SYS -->|"Assigned tasks, Food/disaster requests"| VOL

    ADMIN -->|"Login, Approve/Reject campaigns, Manage users, Assign volunteers"| SYS
    SYS -->|"Dashboard, Users, Campaigns, Tasks"| ADMIN

    GUEST -->|"View campaigns, Browse food donations, Register/Login"| SYS
    SYS -->|"Campaign list, Food donation list"| GUEST
```

## Notes for Mermaid Rendering

- Use a Mermaid-compatible viewer (e.g. https://mermaid.live, GitHub markdown, or VS Code Mermaid extension).
- If a diagram is too wide, wrap the code in a `flowchart TB` and keep external entities at top and processes below, or split into smaller diagrams.
- Data store shape `[(("Label"))]` renders as a cylinder-style node; you can replace with `[(Label)]` if your viewer differs.
- To number processes in Level 1.1–1.5 to match your report, keep the numbering (e.g. 1.2.1, 1.2.2) as-is in the process labels.
